what distinguishes normal HTML elements from semantic HTML elements?

Normal html elements are generic and dont provide any specific meaning,
while Semantic HTML elements have predefined purpose and convey the meaning of the content they wrap.
they help improve accessibility and overall code readability.

Example

Using 'header' tag element instead of 'div' tag element
indicates that it contains the introductory or navigational content of a webpage.

Disparity between closing tags and self-closing tags in HTML

Closing tags are used to close html elements that has an
opening tag, while self-closing tags are used for elements that
dont have any content within them.

Example

The 'p' tag element has both an opening 'p' tag and closing tag '/p'
while the 'img' tag element is self-closing. No closing tag required.
examples of self-closing tags are; 'br', 'hr', 'input' etc.

What distinguishes inline elements from block elements

Inline elements

  • Dont start on a new line.
  • Only take up as much width as necessary to fit their content.
  • Examples include; 'span', 'a', 'em' and 'strong'.

Block elements

  • Start on new line
  • Take upthe full width of their parent container by default.
  • Examples include; 'div', 'p', 'h1'-'h6', 'ul' etc.

these distinctions affect how elements are displayed and interact with
other elements on the page.
